AI Meeting Notes for Obsidian

Record and transcribe meetings inside your vault, 100% locally. This is the original plugin - now superseded by the desktop app.

Get the desktop app instead Plugin on GitHub

⚠️ Deprecated - not recommended. The desktop apps (macOS & Windows) now do everything this plugin does - record, transcribe, and summarize - with zero audio setup (no loopback device) and a built-in meetings library. They write the same Markdown notes into any folder, including an Obsidian vault. This plugin is kept for the record and for the pure-in-Obsidian workflow, but isn't actively developed.

What it does

Captures your microphone and the call's system audio inside Obsidian, transcribes them locally with whisper.cpp, and writes a timestamped Markdown note into your vault. Optional AI summary via a local Ollama model.

Mic ─┐ ├─► Web Audio mixer ─► MediaRecorder ─► OfflineAudioContext Sys ─┘ (loopback) (resample → 16 kHz mono WAV) │ whisper.cpp (whisper-cli) │ transcript ─► Markdown note

Transcription is batch: record the whole meeting, then transcribe on stop. Desktop Obsidian only (it needs to run a local binary and capture audio).

Prerequisites (macOS)

The plugin orchestrates two external pieces you install once - it does not bundle them.

1. A loopback device for system audio

Capturing system audio on macOS needs a loopback (virtual audio) device - install any one you prefer, then select it as the System audio input in the plugin's settings. So you can still hear the call, open Audio MIDI Setup, create a Multi-Output Device containing both your speakers/headphones and the loopback device, and select it as your system output. (Recording just your own mic? Skip the loopback device.)

2. whisper.cpp + a model

brew install whisper-cpp
mkdir -p ~/models
curl -L -o ~/models/ggml-base.en.bin \
  https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-base.en.bin

Install & set up

Via BRAT

  1. Install the BRAT community plugin.
  2. BRAT → Add Beta Plugin → servika/ai-meeting-notes.
  3. Enable AI Meeting Notes under Settings → Community plugins.

Then, in Settings → AI Meeting Notes

  1. Click Grant to allow microphone access (so device names appear).
  2. Choose your Microphone and your System audio (loopback) device.
  3. Set Model path to your .bin file (e.g. ~/models/ggml-base.en.bin).
  4. The whisper-cli binary defaults to whisper-cli; override with a full path if needed.

📖 The step-by-step usage guide walks through the macOS audio routing - the part people get stuck on.
